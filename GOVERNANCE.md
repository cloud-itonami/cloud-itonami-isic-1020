# Governance

This project is maintained by the cloud-itonami team.

## Roles

- **Maintainers**: Oversee architecture, review contributions, release versions.
- **Contributors**: Propose features, report issues, submit pull requests.

## Decision-Making

Significant changes (new operations, protocol modifications, scope expansions)
require maintainer consensus and are documented as architectural decision
records (ADRs).

## Contribution Process

1. Fork the repository
2. Create a feature branch
3. Submit a pull request with a clear description
4. Respond to feedback
5. Upon approval, your contribution is merged

## Code Standards

- All source is `.cljc` (portable, cljs-first)
- No JVM-only constructs
- Tests required for all new functionality
- Documentation must accompany API changes

## Reporting Issues

Use GitHub Issues to report bugs, suggest features, or discuss architecture.
For security issues, see `SECURITY.md`.
